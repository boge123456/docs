# OpenAI 接口 - 视频生成（/v1/videos）

OpenAI 兼容的视频生成入口会将通用字段与不同模型的扩展字段合并成 `OpenAIVideoGenerationRequest`，扩展结构在 `services/provider/biz/chat/entity/video.go` 中的 `ModelRequestExtensions` 里定义，分别对应即梦（`JimengExtensions`）和豆包（`DoubaoExtensions`）的专有能力。不同模型只接受自身列出的字段，传入未支持的参数会得到 400。

## 参数支持矩阵

| 参数 | 说明 | `sora-2` (OpenAI) | `jimeng_ti2v_v30*` | `doubao-seedance*` |
| --- | --- | --- | --- | --- |
| `model` | 指定模型名称 | Yes | Yes | Yes |
| `prompt` | 文本提示词 | Yes | Yes | No |
| `size` | 输出分辨率（如 `1280x720`） | Yes | No | No |
| `seconds` | 时长（OpenAI 风格字符串） | Yes | No | No |
| `input_reference` | multipart 参考文件 | Yes | No | No |
| `content[]` | 富媒体输入数组（文本/首尾帧） | No | No | Yes |
| `return_last_frame` | 返回尾帧图片 | No | No | Yes |
| `resolution` | 预设分辨率（`480p/720p/1080p`） | No | No | Yes |
| `ratio` | 宽高比（`16:9` 等） | No | No | Yes |
| `duration` | 时长（秒，整型） | No | No | Yes |
| `framespersecond` | 帧率设置 | No | No | Yes |
| `camerafixed` | 是否固定运镜 | No | No | Yes |
| `watermark` | 是否带水印 | No | No | Yes |
| `req_key` | 即梦服务标识（必须与模型匹配） | No | Yes | No |
| `binary_data_base64[]` | 图片 Base64 参考 | No | Yes | No |
| `image_urls[]` | 图片 URL 参考 | No | Yes | No |
| `seed` | 随机种子 | No | Yes | No |
| `frames` | 生成帧数（121/241） | No | Yes | No |
| `aspect_ratio` | 即梦长宽比（如 `16:9`） | No | Yes | No |
| `req_json` | 即梦隐式标识 | No | Yes | No |

> 说明：表格中的 `jimeng_ti2v_v30*` 包含 `jimeng_ti2v_v30_pro`、`jimeng_t2v_v30`、`jimeng_t2v_v30_1080p` 等系列；`doubao-seedance*` 包含 `doubao-seedance1.0`、`doubao-seedance-pro` 等型号。豆包模型使用 `content` 数组描述文本及首/尾帧信息，因此不接受 `prompt` 字段；OpenAI `sora-2` 则只支持通用字段，不需要扩展参数。
